  <name>Task 1: Implement NLM helper functions and notebook management commands</name>
  <files>src/commands/research.js, src/router.js, src/lib/constants.js</files>
  <action>
Add the following to `src/commands/research.js`, after the existing `cmdResearchCollect` function:

**1. getNlmBinary(cwd) — Binary resolution helper:**
Calls existing `detectCliTools(cwd)` and checks `notebooklm-py` availability. Returns `{ available: true, path: "..." }` or `{ available: false, error: "...", install_hint: "pip install \"notebooklm-py[browser]\" && playwright install chromium && notebooklm login" }`. Use the detected path (which already handles config.nlm_path override, notebooklm-py binary, and nlm fallback from Phase 56).

**2. checkNlmAuth(nlmPath, timeout) — Auth health probe:**
Runs `execFileSync(nlmPath, ['list', '--json'], { encoding: 'utf-8', timeout: timeout || 10000, stdio: 'pipe' })`. This is the cheapest NLM operation that validates auth cookies.
- On success (JSON.parse succeeds): return `{ authenticated: true, notebooks: parsedResult }`.
- On error: inspect `err.message` and `err.stderr` for auth indicators (`auth`, `cookie`, `login`, `401`, `403`, `session`, `expired`, `unauthorized`). Return `{ authenticated: false, reason: isAuthError ? 'auth_expired' : 'unknown_error', message: ..., reauth_command: 'notebooklm login' }`.
- Wrap the entire function in try/catch — never throw.

**3. formatNlmError(data) — TTY error formatter for NLM commands:**
Simple formatter that shows error message in red, install_hint or reauth_command in yellow if present. Follow the pattern from `formatYtSearch` error handling.

**4. cmdResearchNlmCreate(cwd, args, raw) — Create notebook command:**
- Call `getNlmBinary(cwd)` — if unavailable, return structured error via `output()`.
- Call `checkNlmAuth(nlmPath, authTimeout)` — if not authenticated, return auth error via `output()`.
- Parse positional arg as notebook title (required — error if missing).
- Execute: `execFileSync(nlmPath, ['create', title, '--json'], { encoding: 'utf-8', timeout: 30000, stdio: 'pipe' })`.
- Parse JSON output. Return structured result: `{ notebook_id: parsed.id, title: parsed.title, raw_output: parsed }`.
- On subprocess error: return `{ error: 'Failed to create notebook', details: err.message }`.
- Add TTY formatter `formatNlmCreate(data)` using banner('NotebookLM Create') + result display.
- Use `output(result, { formatter: formatNlmCreate, raw })`.

**5. cmdResearchNlmAddSource(cwd, args, raw) — Add source command:**
- Call `getNlmBinary(cwd)` — if unavailable, return structured error.
- Call `checkNlmAuth(nlmPath, authTimeout)` — if not authenticated, return auth error.
- Parse args: first positional is notebook_id (required), second is source URL/path (required).
- First set active notebook: `execFileSync(nlmPath, ['use', notebookId], { encoding: 'utf-8', timeout: 10000, stdio: 'pipe' })`.
- Then add source: `execFileSync(nlmPath, ['source', 'add', sourceUrl, '--json'], { encoding: 'utf-8', timeout: 60000, stdio: 'pipe' })`. Use 60s timeout — source processing (fetch + index) can be slow.
- Parse JSON output. Return structured result.
- On subprocess error: return structured error (don't crash).
- Add TTY formatter `formatNlmAddSource(data)`.

**6. Wire into router.js:**
In the research namespace case block (around line 700), add:

} else if (subCmd === 'nlm-create') {

Update the error message to include all NLM commands in available subcommands list.

**7. Add COMMAND_HELP entries** in `src/lib/constants.js` for both `research:nlm-create` and `research:nlm-add-source` (and space-separated variants `research nlm-create` etc.).

**8. Export** `cmdResearchNlmCreate` and `cmdResearchNlmAddSource` from module.exports.

**Auth timeout:** Use `Math.min(10000, (loadConfig(cwd).rag_timeout || 30) * 1000 / 3)` for auth check — never more than 10s, fraction of total budget.

**Important:** All NLM commands follow the same guard pattern: check binary → check auth → execute. If either guard fails, return structured JSON error, never crash. This matches the yt-dlp graceful degradation pattern from Phase 57.
