<objective>
Establish one lean config contract for model settings before touching runtime resolution or settings UX.

Purpose: current code mixes `model_profile`, `model_profiles`, and `model_overrides`, which leaves CLI, plugin, docs, and runtime behavior out of sync. This plan defines the smallest shared contract that solves the real user need: keep the global profile selector, let each built-in profile point at a chosen concrete model, and allow direct agent overrides.
Output: shared config defaults, normalization, plugin parsing, and validation all agree on one light project-default-first settings shape.
</objective>

> **TDD Decision:** Selected - normalization and validation behavior is pure, testable input/output logic with clear expected outcomes.
