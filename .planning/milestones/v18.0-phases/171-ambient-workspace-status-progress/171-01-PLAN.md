<objective>
Create the shared ambient snapshot contract that later `cmux` writes can trust.

Purpose: Phase 171 should not scatter sidebar vocabulary or trust gates across plugin hooks. This plan defines one pure, testable mapping from existing project state and recent plugin signals into the user-facing workspace state, compact context, and progress modes that the sidebar will show.
Output: a plugin-local snapshot helper with deterministic precedence, workflow-first context fallback, and explicit progress trust classification.
</objective>

  <behavior>
    Attached `cmux` sessions can derive one trustworthy snapshot that answers three questions: what state should the workspace show, what compact context should accompany it, and whether progress is exact, activity-only, or hidden.
    Cases: ready-to-plan or equivalent human-gated truth -> `Input needed`; true blockers -> `Blocked`; warning-only overlays stay `Warning`; exact numeric progress -> `exact`; active work without earned counts -> `activity`; weak or stale progress -> `hidden`.
  </behavior>
  <implementation>
    Add one plugin-local helper that consumes existing parsed project state, session continuity, blockers, and recent notification severity without making any `cmux` writes itself. Keep task-level context conservative: prefer workflow labels such as `Planning`, `Executing`, or `Verifying`, then degrade to structural phase/plan labels or hide the field instead of treating heuristics as live truth.
  </implementation>
</feature>
