<objective>
Wire the shared ambient snapshot into attached `cmux` sidebar updates without disturbing the rest of the plugin.

Purpose: once the state model exists, Phase 171 still needs one narrow integration path that publishes or clears sidebar metadata at the right times. This plan keeps the write path centralized, reuses the Phase 170 adapter boundary, and ensures stale progress or context never lingers after the underlying signal loses trust.
Output: attached-session sidebar sync that refreshes on trusted lifecycle transitions while suppressed sessions stay quiet and unchanged.
</objective>

> **TDD Decision:** Selected - attached versus suppressed writes, stale metadata clearing, and lifecycle refresh behavior are deterministic integration contracts with clear observable outcomes.
