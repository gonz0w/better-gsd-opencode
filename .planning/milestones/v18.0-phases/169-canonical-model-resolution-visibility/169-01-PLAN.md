<objective>
Create one reusable model-state presenter before updating individual command surfaces.

Purpose: Phase 168 already established canonical resolution, but users still see uneven configured-versus-resolved visibility because each surface exposes slightly different fields. This plan makes helper, decision, enrichment, and misc command output agree on one explicit model-state contract so later init and routing work can consume the same shape.
Output: shared helper-backed model-state output used by decision payloads, command enrichment, `util:resolve-model`, and settings-facing diagnostics.
</objective>

> **TDD Decision:** Selected - the helper output shape and command payloads are deterministic input/output behavior that should be locked by tests before the refactor lands.
