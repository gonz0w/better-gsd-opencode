<objective>
Remove the last provider-tier routing assumptions from orchestration.

Purpose: Phase 169 fails if users can change the concrete models behind shared profiles but routing still behaves differently because it compares `haiku`, `sonnet`, or `opus`. This plan rewrites routing to recommend the public shared profiles first and to resolve concrete models only through the canonical config path.
Output: provider-agnostic orchestration logic and regression coverage that stays stable when profile-backed concrete models change.
</objective>

> **TDD Decision:** Selected - routing recommendations and final resolved-model outcomes are deterministic and should be specified in tests before replacing the provider-tier logic.
