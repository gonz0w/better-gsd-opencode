<objective>
Finish the strict attachment gate with reversible write-path proof and safe no-op methods for later phases.

Purpose: proving the right workspace is not enough if writes can still fail or leak. This plan adds one reserved reversible write probe, marks the adapter attached only after that probe succeeds, and exposes inert methods so later cmux UX phases can build on one trusted boundary instead of redoing safety checks.
Output: a session-sticky write-proven cmux adapter that stays quiet when proof fails and is safe for later status/progress/log calls.
</objective>

> **TDD Decision:** Selected - write-probe success/failure, one-time caching, and inert method behavior are deterministic contracts that can be specified before implementation.
