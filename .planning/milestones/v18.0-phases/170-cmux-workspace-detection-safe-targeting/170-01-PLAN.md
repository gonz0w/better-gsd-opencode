<objective>
Create the narrow cmux transport and adapter contract that all later workspace UX work will depend on.

Purpose: Phase 170 should not spread cmux logic across the plugin before safe targeting exists. This plan establishes one bounded cmux runner, one explicit availability verdict, and one inert adapter boundary so the plugin can stay quiet and unchanged when cmux is absent or unusable.
Output: a plugin-local cmux runner plus a cached no-op-or-available adapter contract wired into plugin startup.
</objective>

> **TDD Decision:** Selected - availability classification, suppression reasons, and plugin fail-open startup are deterministic behavior contracts that should be locked by tests before the adapter exists.
