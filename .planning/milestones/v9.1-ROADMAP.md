# Milestone v9.1: Performance Acceleration & Plugin Benchmarking

**Status:** ✅ SHIPPED 2026-03-10
**Phases:** 77-81
**Total Plans:** 13

## Overview

Dependency-driven performance acceleration for the plugin and CLI by modernizing validation, speeding file discovery, improving startup/cache runtime paths, and shipping rollback-safe controls that preserve existing behavior.

## Phases

### Phase 77: Validation Engine Modernization

**Goal**: Users run plugin tools with lower validation overhead while seeing the same outputs and fallback safety.
**Depends on**: Phase 76 (completed)
**Plans**: 3 plans

Plans:

- [x] 77-01: Create validation adapter module with valibot implementation and zod fallback
- [x] 77-02: Migrate plugin validation to use shared adapter with fallback parity
- [x] 77-03: Wire adapter into bgsd_context and bgsd_progress tools

**Details:**
- Default plugin validation path now uses valibot with explicit zod fallback flags
- Adapter error normalization keeps invalid-input output contracts identical across validation engines
- Recorded VALD-01 baseline evidence with 34.48% modern-path improvement against forced fallback

### Phase 78: File Discovery and Ignore Optimization

**Goal**: File-heavy workflows complete faster while preserving exact legacy file-selection behavior.
**Depends on**: Phase 77
**Plans**: 3 plans

Plans:

- [x] 78-01: Create dual-path file discovery adapter with fast-glob integration
- [x] 78-02: Implement in-process ignore matching to eliminate subprocess overhead
- [x] 78-03: Add parity verification and diagnostics for file discovery

**Details:**
- Discovery hotspots now route through a dual-path adapter — Enable fast-glob and in-process ignore rollout behind a parity-safe legacy default
- Optimized discovery is now the default runtime path — legacy mode available via BGSD_DISCOVERY_MODE=legacy for diagnosis
- Source-dir detection is fully equivalent between legacy and optimized

### Phase 79: Startup Compile-Cache Acceleration

**Goal**: Repeated CLI invocations start faster through guarded compile-cache usage without runtime breakage.
**Depends on**: Phase 78
**Plans**: 3 plans

Plans:

- [x] 79-01: Add compile-cache guard to CLI with environment variable and config support
- [x] 79-02: Create bin/bgsd wrapper script for warm-start optimization
- [x] 79-03: Fix compile-cache flag regression on Node 22+ (gap closure)

**Details:**
- Compile-cache guard uses BGSD_COMPILE_CACHE env var (default disabled for safety)
- Runtime capability detection identifies Node 10.4+ support, graceful fallback on older runtimes
- RUNT-01 achieved - warm starts are faster (76-102ms) due to Node 22+ compile-cache by default

### Phase 80: SQLite Statement Cache Acceleration

**Goal**: Cache-heavy command paths show lower tail latency through SQLite statement caching with compatibility fallback.
**Depends on**: Phase 79
**Plans**: 1 plan

Plans:

- [x] 80-01: Implement SQLite statement caching in CacheEngine

**Details:**
- Statement caching via createTagStore() reduces p50 latency by ~43%, p99 by ~22%
- BGSD_SQLITE_STATEMENT_CACHE env var controls feature (default: enabled on Node 22.5+)

## Milestone Summary

### Key Decisions

- v9.1 roadmap: Scope is dependency-driven acceleration only; benchmark-framework expansion remains out of scope
- v9.1 roadmap: Requirements are grouped into five delivery phases (validation, scan/ignore, startup, cache, safety/parity)
- v9.1 roadmap: Backward compatibility and rollback controls are treated as explicit milestone outcomes, not implicit assumptions
- Phase 77: Default plugin validation path now uses valibot with explicit zod fallback flags
- Phase 78: Optimized discovery is now the default runtime path
- Phase 81: Created generalized parity-check utility and util:parity-check command

### Issues Resolved

- Fixed context overflow at high phase counts
- Eliminated git check-ignore subprocess overhead in file discovery
- Reduced cache-layer tail latency via SQLite statement caching
- Added rollback-safe controls for all dependency-backed optimizations

### Issues Deferred

- Benchmark framework expansion (moved to future milestone)
- CLI tool integrations (ripgrep, fd, fzf, etc.) - see REQUIREMENTS.md backlog
