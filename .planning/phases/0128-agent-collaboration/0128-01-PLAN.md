  <action>
Add two new decision functions after the Phase 127 tool routing section (after line ~756):

**`resolveAgentCapabilityLevel(state)`:**
- Input: `{ tool_availability: {ripgrep, fd, jq, yq, bat, gh} }`
- Count how many of the 6 tools are `true` in tool_availability
- Return thresholds per CONTEXT.md decision:
  - 5-6 tools → `{ value: 'HIGH', confidence: 'HIGH', rule_id: 'agent-capability-level' }`
  - 2-4 tools → `{ value: 'MEDIUM', confidence: 'HIGH', rule_id: 'agent-capability-level' }`
  - 0-1 tools → `{ value: 'LOW', confidence: 'HIGH', rule_id: 'agent-capability-level', metadata: { warning: 'Low tool availability — agent proceeding with limited capabilities' } }`
- Only LOW triggers warning metadata (per CONTEXT.md: MEDIUM is normal, no behavior change)
- Capability score has NO connection to model profile selection (independent concern)

**`resolvePhaseDependencies(state)`:**
- Input: `{ phases: [{ number, depends_on: [], tool_requirements: [], status }], tool_availability: {} }`
- Algorithm:
  1. Build dependency graph from declared `depends_on` arrays
  2. Topological sort — declared dependencies ALWAYS WIN (per CONTEXT.md)
  3. For phases with NO declared dependency relationship (both are roots or at same level), use tool availability as tiebreaker: phases whose required tools are available go first
  4. Built-in heuristic: if phase name contains 'discovery'/'detect'/'scan', prioritize before 'analysis'/'transform'/'complex' phases at the same dependency level
  5. If conflict (dependency order vs tool readiness): honor dependencies, include `metadata: { warning: 'Phase N has suboptimal tool readiness' }` for any phase missing required tools
- Return: `{ value: { ordered_phases: [numbers], warnings: [] }, confidence: 'HIGH'|'MEDIUM', rule_id: 'phase-dependencies' }`
- confidence: HIGH if all dependencies resolved cleanly, MEDIUM if warnings present

Register both in `DECISION_REGISTRY` array:
- `{ id: 'agent-capability-level', name: 'Agent Capability Level', category: 'state-assessment', description: 'Scores agent capability based on available tool count', inputs: ['tool_availability'], outputs: ['HIGH|MEDIUM|LOW'], confidence_range: ['HIGH'], resolve: resolveAgentCapabilityLevel }`
- `{ id: 'phase-dependencies', name: 'Phase Dependency Sequencing', category: 'workflow-routing', description: 'Sequences phases accounting for dependencies and tool capabilities', inputs: ['phases', 'tool_availability'], outputs: ['{ ordered_phases, warnings }'], confidence_range: ['HIGH', 'MEDIUM'], resolve: resolvePhaseDependencies }`

Add both to `module.exports`.
