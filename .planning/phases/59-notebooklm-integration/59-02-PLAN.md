Add the following to `src/commands/research.js`, after the Plan 01 NLM functions:

**1. cmdResearchNlmAsk(cwd, args, raw) — Ask question against notebook:**
- Guard: `getNlmBinary(cwd)` → `checkNlmAuth(nlmPath, timeout)` → proceed.
- Parse args: first positional is notebook_id (required), remaining positional args joined as question (required).
- Parse `--new` flag (starts fresh conversation if present).
- Set active notebook: `execFileSync(nlmPath, ['use', notebookId], { timeout: 10000, ... })`.
- Ask question: `execFileSync(nlmPath, ['ask', question, '--json', ...(newFlag ? ['--new'] : [])], { timeout: 30000, ... })`.
- Parse JSON output. Return: `{ notebook_id, question, answer: parsed.answer, references: parsed.references || [], raw_output: parsed }`.
- On error: return `{ error: 'Failed to ask question', details: err.message }`.
- Add TTY formatter `formatNlmAsk(data)` — show question in bold, answer text, and references list.

**2. cmdResearchNlmReport(cwd, args, raw) — Generate report:**
- Guard: `getNlmBinary(cwd)` → `checkNlmAuth(nlmPath, timeout)` → proceed.
- Parse args: first positional is notebook_id (required). Parse `--type` flag (default: `briefing-doc`). Valid types: `briefing-doc`, `study-guide`, `blog-post`.
- Parse optional `--prompt` flag for custom report prompt.
- Set active notebook: `execFileSync(nlmPath, ['use', notebookId], { timeout: 10000, ... })`.
- Generate report: `execFileSync(nlmPath, ['generate', 'report', '--type', reportType, '--json', ...(customPrompt ? ['--prompt', customPrompt] : [])], { timeout: 60000, ... })`. Use 60s timeout — report generation is slow.
- Parse JSON output. Return structured result with report content.
- On error: return `{ error: 'Failed to generate report', details: err.message }`.
- Add TTY formatter `formatNlmReport(data)`.

**4. collectNlmSynthesis(cwd, query, sources, timeout) — Tier 1 helper:**
This internal function orchestrates the NLM synthesis pipeline:
- Call `getNlmBinary(cwd)` — if unavailable, return null (silent fallback).
- Call `checkNlmAuth(nlmPath, 5000)` — if not authenticated, return null (silent fallback).
- Create notebook: `execFileSync(nlmPath, ['create', `[GSD] ${query.slice(0, 50)}`, '--json'], { timeout, ... })`.
- Extract notebook_id from result.
- Add top 2-3 source URLs to notebook (only web URLs and YouTube URLs, skip sources without URLs):
  ```
  for (const src of sources.filter(s => s.url).slice(0, 3)) {
    execFileSync(nlmPath, ['use', notebookId], ...);
    execFileSync(nlmPath, ['source', 'add', src.url, '--json'], { timeout, ... });
  }
  ```
- Ask synthesis question: `execFileSync(nlmPath, ['use', notebookId], ...); execFileSync(nlmPath, ['ask', `Provide a comprehensive technical summary about: ${query}. Focus on practical implementation details, best practices, and common pitfalls.`, '--json', '--new'], { timeout, ... })`.
- Parse answer. Return `{ answer: parsed.answer, notebook_id, sources_loaded: N }`.
- Wrap ENTIRE function in try/catch. On ANY error, log via `debugLog('research.nlm', ...)` and return null. **Never crash the pipeline.** NLM failure = silent fallback to Tier 2 sources.
