**TTY Formatter:**
Add a `formatAuditScan(data)` function to `src/commands/audit.js` following the output formatter pattern from other command modules (see `formatStateShow` pattern in `src/commands/state.js` or codebase command formatters).

The formatter should produce:
1. **Banner:** `bGSD ▶ AUDIT SCAN` using `banner()` from format.js
2. **Summary section:** Using `sectionHeader('Summary')` — show total candidates, offloadable count, keep count, estimated total savings
3. **Offloadable candidates table:** Using `formatTable()` — columns: ID, Category, Rubric Score (e.g., "3+4/7"), Token Savings/Session, Priority. Sorted by total_score descending, then by per-session savings descending.
4. **Keep in LLM section:** Using `sectionHeader('Keep in LLM')` — list each candidate with its ID, category, which critical criteria failed, and rationale. Use dim text for these entries.
5. **Savings by category table:** Using `formatTable()` — columns: Category, Candidate Count, Total Savings/Session. Show which categories contribute most savings.
6. **Action hint:** Using `actionHint()` — suggest next step: "Run /bgsd plan phase 111 to build the decision engine"

Wire the formatter into the `cmdAuditScan` handler using the migrated output pattern:
```javascript
output(result, {
  formatter: formatAuditScan
});
```

**Catalog artifact generation:**
Add a `--save` flag (or make it default) that writes the scan results to `.planning/audit-catalog.json` in the project's `.planning/` directory. The JSON file should contain:
- `generated_at`: ISO timestamp
- `scanner_version`: "1.0.0"
- `files_scanned`: count of workflow + agent files scanned
- `candidates`: full array with all rubric scores and token estimates
- `offloadable`: filtered array of passing candidates
- `keep_in_llm`: filtered array of failing candidates with rationale
- `summary`: aggregate statistics
- `token_model`: description of the estimation methodology

This artifact is the primary input for Phase 111 (Decision Engine).

After writing, emit a status message: `status('Catalog written to .planning/audit-catalog.json')`.
